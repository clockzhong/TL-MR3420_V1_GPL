1	用途：本程序在于去除web页面中的“<!--<TP_C_TAG>-->”等标志及其之前的所有文字，并将修改后的文件拷贝至指定的目录下。
2	使用说明：
	配置：首先在config文件中添加TAG,每行一个TAG，TAG之后即换行，不要有其他字符。例如：
		<!--<TP_C_TAG>-->
		<!--<TP_C_TAG>//-->
		
	命令： ./webc -s path1:path2:...  -d path3
	选项说明：
			-s 或者 -S		指定所要修改的文件所在目录，如果源文件分布在多个目录中，则使用“:”将这若干个目录连接起来；
			-d 或者 -D		指定修改后的文件所要拷贝至的目录，只能有唯一一个目的目录。如果该目录不存在，程序将会建立。
	
	使用示例：
			1、 ./webc -s ./web -d ./dst	将当前目录下的子目录web中所有的文件拷贝至dst目录下再去除TAG。这儿使用的是相对路径，也可以使用绝对路径；
			2、 ./webc -s ./web:/root/yuchuwei/oem -d ./dst	将web目录和oem目录中的文件拷贝至dst目录再去除TAG。
			3、 ./webc -s ./web	对web目录下的文件进行修改，但不拷贝。
			
	注意：
	1 本程序能够循环访问一个目录下的所有子目录和文件;
	2 每一个目录路径，包括源路径和目的路径，最后直接写目录名称即可，不要加“/”。例如我们的源路径或者目的路径是“/root/web”，千万不要写成“/root/web/”，
	否则后果自负，特此声明。
